### RTT 的定义

通常意义上来讲，RTT 是指数据包从发送端到接收端的时间，在 Wireshark 的 TCP RTT 图中，当抓包所处位置不同时，RTT 有不同的定义：

1. 抓包点位于 sender，我们命名为 RTT1：RTT = 数据包发出时间 与 ACK 收到时间之间的差值
2. 抓包点位于 receiver，我们命名为 RTT2：RTT = 数据包抵达时间 与 ACK 发出时间的差值

上面的定义在 Wireshark 中需要对应数据方向，

RTT1：需要抓 sender 侧，数据方向为 sender -> receiver
RTT2：需要抓 receiver 侧，数据方向为 receiver -> sender

### BDP

BDP = 瓶颈带宽 \* 最小延迟

BDP 决定了在网络中正在传输的数据（in-flight data）最多是多少（不排队的前提，也就是说不适用 Buffer）

## 缓存

### MAC 层缓存（发送/接收缓存）

```shell
# 安装查看工具
apt install ethtool
# 查看缓存
ethtool -g <interface>
# 设置缓存
ethtool -G <interface> <tx> <rx>
```

### 协议栈缓存

```shell
# 查看协议栈缓存
sysctl -a | grep net.ipv4.tcp_
# 设置协议栈缓存
sysctl -w net.ipv4.tcp_wmem="4096 16384 4194304" # 发送缓存:最小值, 默认值, 最大值
sysctl -w net.ipv4.tcp_rmem="4096 87380 33554432" # 接收缓存:最小值, 默认值, 最大值
```

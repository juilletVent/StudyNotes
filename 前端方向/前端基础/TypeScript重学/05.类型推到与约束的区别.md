## 类型推断与约束的区别

**重点哲学：类型约束多数场景下只用于验证，不用于推导，部分场景下参与类型推导（any、any[]、object）**

官方案例：

```typescript
function firstElement1<Type>(arr: Type[]) {
  return arr[0];
}

function firstElement2<Type extends any[]>(arr: Type) {
  return arr[0];
}

// a: number (good)
const a = firstElement1([1, 2, 3]);
// b: any (bad)
const b = firstElement2([1, 2, 3]);
```

首先看第一个案例，firstElement1 没有任何问题，返回值类型被推断为 number，根据实际参数入参，可以推导Type为number，虽然没有显示声明返回值，但是可以很明确的根据索引访问的方式: Type[0]，推导出返回值类型为number。

Ok，那么第二个案例呢？

实际参数类型为：number[]，赋值给类型 Type，检查类型约束 Type extends any[]，number[] 是 any[] 的子类型，所以类型约束通过，然后尝试推导返回值类型，由于约束条件使用了 any[]，这会影响类型推导（大部分场景下，类型约束并不会影响类型推导），造成类型放宽（widening），因此arr[0]的类型被推导为 any，而不是 number。核心原因是 any以及与any有关的类型是吞噬者类型，它们会吃掉任何类型，造成类型放宽。

再来看一个案例：

```typescript
function firstElement3<Type extends T[], T>(arr: Type): T {
  return arr[0];
}

// c: unknown
const c = firstElement3([1, 2, 3]);
```

这个案例更进一步，同样的，根据实际参数，Type被推导为 number[]，检查类型约束 Type extends T[]，满足约束，继续尝试推导 T，发现无法从现有信息中推导出T，因此T取默认类型：unknown，继续推导 arr[0]，最终返回值类型为 unknown。

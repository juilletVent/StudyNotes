## 基本概念

“声明合并”指的是编译器将两个同名声明合并成一个定义。合并后的定义将同时具备两个原始声明的特性。合并的声明数量不限，不限于两个。

## Interface 合并

接口的非函数成员必须是唯一的。如果不是唯一的，则它们的类型必须相同。如果两个接口都声明了同名但类型不同的非函数成员，编译器将报错。

```ts
interface Box {
  height: number;
  width: number;
}
interface Box {
  scale: number;
}
let box: Box = { height: 5, width: 6, scale: 10 };
```

对于函数成员，同名函数成员均被视为描述同一函数的不同重载版本。此外，值得注意的是，如果接口 A 与后续接口 A 合并，则后合并的接口优先级高于前者。

```ts
interface Cloner {
  clone(animal: Animal): Animal;
}
interface Cloner {
  clone(animal: Sheep): Sheep;
}
interface Cloner {
  clone(animal: Dog): Dog;
  clone(animal: Cat): Cat;
}

// 合并后
interface Cloner {
  clone(animal: Dog): Dog;
  clone(animal: Cat): Cat;
  clone(animal: Sheep): Sheep;
  clone(animal: Animal): Animal;
}
```

_Tips：借助接口合并这个特性，我们可以为已有的库、框架添加自己的类型定义_

## 增强模块

JavaScript 模块不支持合并，但可以通过导入并更新现有对象来对其进行修补：

```ts
// person.ts
export class Person {
  constructor(public name: string) {}

  sayHi() {
    console.log(this.name);
  }
}

// programer.ts
import { Person } from "./person";

// 类型“Person”上不存在属性“sayHello”
Person.prototype.sayHello = function () {
  console.log("Hello world.");
};
```

修补之后TS并不知道map得存在，无法通过类型检查，需要通过额外的模块增强来告知编译器增强函数的存在：

```ts
// programer.ts
import { Person } from "./person";

declare module "./person" {
  interface Person {
    sayHello: () => void;
  }
}

Person.prototype.sayHello = function () {
  console.log("Hello world.");
};
```

此时，使用sayHello就不会报错了：

```ts
import { Person } from "./person";
import "./programer";

const alice = new Person("Alice");

alice.sayHi();
alice.sayHello();
```

_Tips：为全局环境下的某些组件添加增强也是一样的操作方式_

```ts
declare global {
  interface Array<T> {
    toObservable(): Observable<T>;
  }
}
Array.prototype.toObservable = function () {
  // ...
};
```

## 典型坑：提取 Enum 的所有 Key

Enum 与其他类型有一个很大的不同：其它类型在运行时并不会对运行时有什么影响，但是Enum会对运行时产生影响--会产出一个实际的Enum枚举对象

```ts
// ts
enum Animal {
  Cat,
  Fish,
}

// js
("use strict");
var Animal;
(function (Animal) {
  Animal[(Animal["Cat"] = 0)] = "Cat";
  Animal[(Animal["Fish"] = 1)] = "Fish";
})(Animal || (Animal = {}));
```

可以看到，enum 在TS编译后是有实际产出影响的，产出了一个对象，这是非常重要的一个前提！

那么也就意味着：**Enum其实有两种解释--类型 or 值**

首先明确：keyof 操作的对象是类型，不能是值，也就是说 `keyof Animal` 中的 Animal 说的是类型系统中的 `Animal`！

那么我们直接 `keyof Animal` 操作的 `Animal` 是什么玩意儿呢 ?

```ts
// 可以赋值什么？
// const a: Animal = ??;
const a: Animal = Animal.Cat;
const b: Animal = Animal.Fish;
const c: Animal.Cat | Animal.Fish = Animal.Fish;
```

也就是说：TS 类型系统中的 `Animal = Animal.Cat | Animal.Fish` 那 `keyof Animal` 本质上就是：`keyof (0|1)`，显然不对劲：

```ts
// 实际上是在对number类型做keyof
// "toString" | "toFixed" | "toExponential" | "toPrecision" | "valueOf" | "toLocaleString"
type ALL_KEY = keyof Animal;
```

那么我们是否可以借助值类型来反推呢？由于`typeof` 只作用于值，那么我们就可以使用：`typeof Animal` 来反向获取Enum在值类型空间中对应的对象类型了。

```ts
type AnimalEnumType = typeof Animal;
// 验证类型
const obj: AnimalEnumType = {
  Cat: 0,
  Fish: 1,
};
```

此时再搭配`keyof`就可以获取到正确得Key集合了：

```ts
type ALL_KEY = keyof AnimalEnumType;
// 连起来就是
type ALL_KEY = keyof typeof Animal;
```

## TS 对 Enum 类型的特殊处理

### 反向映射

由于TS对Enum编译产出的对象包含反向映射（方便通过值查询Key），因此可以使用这种方式反向访问：

```ts
enum Animal {
  Cat,
  Fish,
}
// string Cat
const typeName = Animal[Animal.Cat];
```

**重点：反向映射仅为number类型enum创建，string类型的enum并不会创建反向映射**

## 类型系统故意忽略

由于TS对Enum编译产出的对象包含反向映射，那么我们 `typeof Animal` 应该得到的类型应该是：

```ts
type AnimalEnumType = {
  Cat: Animal.Cat;
  Fish: Animal.Fish;
  0: string;
  1: string;
};
```

但是测试你会发现，出来的类型中并没有0、1这两个属性，这是TS故意设计的，否则：`keyof typeof Animal` 将得到：`Cat | Fish | 0 | 1`，显然，这肯定不是我们想要的。

## const enum

上面提到enum会在运行时创建相应的枚举对象，为了避免因访问枚举值而产生额外的代码生成成本和间接寻址，可以使用 const 枚举，这会使得TS直接将enum值内联在引用的位置，不生成Enum枚举对象，也就不会产生寻址成本。

```ts
// TS
const enum Animal {
  Cat,
  Fish,
}

const a = Animal.Cat;
const b = Animal.Fish;

// js
("use strict");
const a = 0; /* Animal.Cat */
const b = 1; /* Animal.Fish */
```

## 环境枚举

如果全局或外部已经提供了全局上下问的枚举定义，但是没有提供TS类型声明，则可以通过环境枚举来补充类型定义:

```ts
// 假设外部已经提供了全局上下文的枚举定义
window.MyEvent = {
  Ok: 0,
  Failed: 1,
};
```

此时我们就可以通过环境枚举来进行声明：

```ts
declare enum MyEvent {
  Ok,
  Failed,
}
// Ok，编译、运行都不会出错
console.log(MyEvent.Ok);
```

## 对象 vs 枚举

我们可以使用对象+`as const`语法来创建对象映射，搭配`keyof`、`typeof`，同样可以达到Enum的效果：

```ts
const Animal = {
  Cat: 1,
  Fish: 2,
} as const;

type AnimalKey = keyof typeof Animal;
type AnimalValue = (typeof Animal)[keyof typeof Animal];

function sayHello(type: AnimalValue) {
  // ...
}

sayHello(Animal.Cat);
```

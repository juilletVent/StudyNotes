## 减少泛型参数使用

泛型参数使用的原则：如果一个类型不关联多个值，则他就没有存在的必要，观察下面的案例：

```typescript
function filter1<Type>(arr: Type[], func: (arg: Type) => boolean): Type[] {
  return arr.filter(func);
}

function filter2<Type, Func extends (arg: Type) => boolean>(
  arr: Type[],
  func: Func,
): Type[] {
  return arr.filter(func);
}
```

filter1 中Type类型在arr，func 以及返回值中中都会被使用，因此他的存在是合理的。

filter2 中，我们创造了一个新的泛型类型：Func，但是仅仅用于函数类型的检查，只引用了一次，因此这个泛型类型是不必要的，应该被删除。

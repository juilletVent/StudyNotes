## 类型保护

TS类型推断与收窄原则：TS只在确定场景下会进行收窄尝试，在泛型场景下，多个参数的类型TS会尝试求解一个“最大公约数”用作类型定义，而不是一般直觉下的类型约束，并且这个最大公约数的求解过程中是所有泛型参数都参与计算的，并非只有第一个参数或者自左向右求解，这点非常重要，正是由于这种原因，泛型场景下的泛型参数类型很可能会被扩大（widening）。

**重点哲学：类型约束多数场景下只用于验证，不用于推导，部分场景下参与类型推导（any、any[]、object）**

### 类型守卫

使用 if + typeof 语句来缩小类型范围，注意针对null的处理，`typeof null === 'object'`。

```typescript
function padLeft(value: string, padding: string | number) {
  if (typeof padding === "number") {
    return Array(padding + 1).join(" ") + value;
  }
  if (typeof padding === "string") {
    return padding + value;
  }
  throw new Error(`Expected string or number, got '${padding}'.`);
}
```

### 真值守卫

if判断可以排除掉null、undefined、false、0、''等假值字面量类型。

```typescript
type AllType = null | undefined | 0 | "" | false | 1 | "22";
let a: AllType;

if (a) {
  // type: 1 | "22"
  console.log(a);
}
```

### 平等范围缩小

当两个参数进行相等比较时，TypeScript会将它们的类型缩小为相等的类型。

```typescript
function example(x: string | number, y: string | boolean) {
  if (x === y) {
    // 在这个分支中，x 和 y 的类型都被缩小为 string
    console.log(x.toUpperCase()); // 可以安全调用 string 方法
  } else {
    // x: number y : boolean
    console.log(x, y);
  }
}
```

### in 操作符缩小范围

JavaScript 有一个运算符 `in`，用于判断一个对象或其原​​型链是否具有名为 ` in 的属性。TypeScript 会利用这一点来缩小潜在类型的范围。

```typescript
type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird) {
  if ("swim" in animal) {
    // 通过判断属性是否存在，缩小类型范围为 Fish 类型
    return animal.swim();
  }

  return animal.fly();
}
```

### instanceof 操作符缩小范围

与in操作符类似，instanceof操作符也可以用来缩小类型范围。

```typescript
class Dog {
  bark() {
    console.log("Woof!");
  }
}

class Cat {
  meow() {
    console.log("Meow!");
  }
}

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    // 通过判断实例类型，缩小类型范围为 Dog 类型
    animal.bark();
  } else {
    // 在这个分支中，animal 的类型被缩小为 Cat 类型
    animal.meow();
  }
}
```

### Assignments 分配检查

TS会基于控制流分析，自动缩小类型范围，这个行为不会更变变量的初始类型声明，只会根据当前的控制流，将类型范围缩小为可能的类型，如果编译器经历了无法推断的不确定控制流，则不会进行任何类型收窄。

```typescript
let a: string | number | boolean;

a = "hello";
// type: string
a.toUpperCase();
// HELLO

a = 123;
// type: number
a.toUpperCase();
// error: Property 'toUpperCase' does not exist on type 'number'.

a = true;
// type: boolean

a = null;
// error: Type 'null' is not assignable to type 'string | number | boolean'.
```

## Never 与 Unknown

1. Never 类型表示一个不会出现的值，是任何类型的子类型，never可以赋值给任何类型，但是任何类型不能赋值给never。
2. Unknown 表示一个未知的类型，是任何类型的父类型，unknown不可以赋值给任何类型，但是任何类型都可以赋值给unknown。
3. 在泛型、函数参数、条件类型等场景下需要注意函数参数逆变模式带来的约束影响，逆变场景下，never 将会是最宽松的约束，而unknown 却变成最严格的约束，这与平时使用这两个类型时的语义恰恰相反

## In 操作符搭配Keyof使用变更属性类型、Readonly、?修饰符

OptionsFlags 会获取 Type Type 中的所有属性，并将它们的值更改为布尔值

```ts
type Point = {
  x: number;
  y: number;
};

type MapToBoolean<T> = {
  [Key in keyof T]: boolean;
};

type OptPoint = MapToBoolean<Point>;
// type OptPoint = {
//     x: boolean;
//     y: boolean;
// }
```

需要注意：如果映射的原始类型带有：read-only、optional 修饰符，则默认情况下，映射的属性也会带有这些修饰符。

```ts
type Account = {
  readonly id: number;
  readonly name: string;
  readonly desc?: string;
};

type OptAccount = MapToBoolean<Account>;
// type OptAccount = {
//     readonly id: boolean;
//     readonly name: boolean;
//     readonly desc?: boolean;
// }
```

映射期间还可以应用两个额外的修饰符： readonly 和 ? ，它们分别影响可变性和可选性。可以通过在前缀前添加 - 或 + 来移除或添加这些修饰符。如果不添加前缀，则默认为 +

```ts
// 移除 readonly
type MapToMutableBoolean<T> = {
  -readonly [Key in keyof T]: boolean;
};
// 移除 ?
type MapToRequiredBoolean<T> = {
  [Key in keyof T]-?: boolean;
};

// 添加 readonly 与 ? 可以省略 + 也可以不省略
type MapToBoolean<T> = {
  [Key in keyof T]?: boolean;
  // 下面这几种也是可以的
  // [Key in keyof T]+?: boolean;
  // readonly [Key in keyof T]?: boolean;
  // +readonly [Key in keyof T]+?: boolean;
};
```

## In 操作符搭配 as 变更 Key

**重点：as 后的模式是作为匹配模式以及目标Key生成使用，不是断言，这与as断言没有任何关系**

假设：K in T

```ts
// 使用 as 创建特定匹配模式，为Key增加前缀
type Prefix<T> = {
  // 此处 Key 可能是非常多类型：number | string |
  [Key in keyof T as `str_${Key & string}`]: T[Key];
  // 兼容性更好一点也可以这样写，主要原因是TS对象的 Key 类型允许的类型为：string | number | symbol，keyof T 也就返回这些类型组合成的联合类型
  // 然后，在模式中，symbol 是无法被转换为字符串的，所以这里使用 Exclude 移除了 symbol 类型就可以完成了，上面使用类型组合 string 其实也是同样的道理
  // 只是上面的那样做，就只有string类型会被保留，其他类型，number | boolean | symbol 均被移出了
  [Key in keyof T as `str_${Exclude<Key, Symbol>}`]: T[Key];
};

type Point = {
  x: number;
  y: number;
};

type Point2 = Prefix<Point>;

const pstr: Point2 = {
  str_x: 1,
  str_y: 2,
};
```

## 明确的属性过滤

```ts
type ExcludeProperty<T, E> = {
  [Key in keyof T as Exclude<Key, E>]: T[Key];
};

type Point3D = {
  x: number;
  y: number;
  z: number;
};

type Point2D = ExcludeProperty<Point3D, "z">;
```

## 根据Value类型进行属性过滤

使用约束搭配as过滤出符合特定约束的属性

```ts
type OnlyFunction<T> = {
  [K in keyof T as T[K] extends Function ? K : never]: T[K];
};

type NotFunction<T> = {
  [K in keyof T as T[K] extends Function ? never : K]: T[K];
};

type Opt = {
  a: () => void;
  b: (val: number) => number;
  c: string;
  d: number;
};

type OptFn = OnlyFunction<Opt>;
type OptProperty = NotFunction<Opt>;
```

## 复制拆分

as 还可以配合联合类型分发将单个属性复制成多个：

```ts
type AddOld<T> = {
  // 使用联合类型分发特性完成
  [K in keyof T as `old_${Exclude<K, symbol>}` | K]: T[K];
};

type Opt = {
  name: string;
};

type NewOpt = AddOld<Opt>;
/* 
type NewOpt = {
    name: string;
    old_name: string;
}
*/
```

## 一个有用的实际案例：根据事件对象生成对应的事件处理声明

下面的示例，我们定义了一个事件类型Map，然后定义了一个工具类型，根据这个事件类型Map生成对应的事件处理映射集合，这个集合中，事件名称会自动变为大写，并且会自动添加on前缀，这样，我们就可以基于这个集合生成对应的事件处理声明了

并且后续如果新增了事件，我们只需要在事件类型EventMap中添加即可，后续生成的事件处理映射集合中，也会自动添加这个事件处理声明，无需同步修改

```ts
// 事件类型定义Map
type EventMap = {
  click: MouseEvent;
  change: Event;
  submit: SubmitEvent;
};

// 事件处理映射集合工具类型
type GenHandlerMap<T> = {
  [K in keyof T as `on${Capitalize<K & string>}`]: (val: T[K]) => void;
};

// 根据事件类型Map生成对应的事件处理器类型映射
type EventHandlerMap = GenHandlerMap<EventMap>;
/* 
type EventHandlerMap = {
    onClick: (val: MouseEvent) => void;
    onChange: (val: Event) => void;
    onSubmit: (val: SubmitEvent) => void;
}
*/
```

## 对任意联合提进行映射

```ts
type EventConfig<Events extends { kind: string }> = {
  // 提取联合体中的公共 Kind 属性
  // 然后取 Kink 属性的字面量作为Key
  [E in Events as E["kind"]]: (event: E) => void;
};

type SquareEvent = { kind: "square"; x: number; y: number };
type CircleEvent = { kind: "circle"; radius: number };

type Config = EventConfig<SquareEvent | CircleEvent>;
/* 
type Config = {
    square: (event: SquareEvent) => void;
    circle: (event: CircleEvent) => void;
}
*/
```

## 用途

主要服务于切面编程，比如Log、Authorization、Cache、Monitoring、Validation、..等需要横切逻辑的场景。

如果不是横切场景，装饰器的价值就不大了。

## 一个方法装饰器基本样例

```ts
// 装饰器
function A(message: string) {
  console.log("calc AAA decorators");

  return function (
    target: any,
    propertyKey: string,
    descriptor: PropertyDescriptor,
  ) {
    console.log("decorator AAA called with message:", message);
  };
}

function B(message: string) {
  console.log("calc BBB decorators");

  return function (
    target: any,
    propertyKey: string,
    descriptor: PropertyDescriptor,
  ) {
    console.log("decorator BBB called with message:", message);
  };
}

class Person {
  constructor(public name: string) {}

  @A("hello")
  @B("world")
  sayMyName() {
    console.log(this.name);
  }
}

new Person("heisenberg").sayMyName();
/* 
calc AAA decorators
calc BBB decorators
decorator BBB called with message: world
decorator AAA called with message: hello
heisenberg
*/
```

## 分类

常用装饰器分为四类：ClassDecorator、MethodDecorator、PropertyDecorator、ParameterDecorator、AccessorDecorators

### ClassDecorator

典型用途：冻结类、注册元数据、添加静态信息

装饰器影响运行时，不影响静态类型，因此如果装饰器需要包装一个类，那么包装后的类签名需要保持一致否则会报类型错误。

```ts
function reportableClassDecorator<T extends new (...arg: any[]) => {}>(
  constructor: T,
) {
  return class extends constructor {
    reportingURL = "http://www...";
  };
}

@reportableClassDecorator
class BugReport {
  type = "report";
  title: string;

  constructor(t: string) {
    this.title = t;
  }
}

const bug = new BugReport("Need more documentation!");
console.log(bug.title);
console.log(bug.type);

// TS类型检查错误，但是如果直接忽略检查，直接编译运行的话，则正常
// 类型“BugReport”上不存在属性“reportingURL”。
console.log(bug.reportingURL);
```

### MethodDecorator

方法装饰器声明在方法声明之前。该装饰器应用于方法的属性描述符 ，可用于**观察、修改或替换方法定义**。

```ts
class Greeter {
  greeting: string;
  constructor(message: string) {
    this.greeting = message;
  }

  @enumerable(true)
  greet() {
    return "Hello, " + this.greeting;
  }
}

function enumerable(value: boolean) {
  return function (
    target: any,
    propertyKey: string,
    descriptor: PropertyDescriptor,
  ) {
    descriptor.enumerable = value;
  };
}

const greeter = new Greeter("world");

console.log(greeter.greet());

// 方法位于原型对象上，默认是不可枚举的，使用装饰器改变枚举性后，可以枚举出来
console.log(Object.keys(Object.getPrototypeOf(greeter)));
// 查看属性的具体配置情况，可以直观的观察到枚举性质被设置为了true
console.log(
  Object.getOwnPropertyDescriptor(Object.getPrototypeOf(greeter), "greet"),
);
```

另外一个具备实际意义的案例：

```ts
class Greeter {
  greeting: string;
  constructor(message: string) {
    this.greeting = message;
  }

  @measure
  greet() {
    return "Hello, " + this.greeting;
  }
}

function measure(
  target: any,
  propertyKey: string,
  descriptor: PropertyDescriptor,
) {
  // 保存原始方法
  const originalMethod = descriptor.value;

  // 包装并替换原始方法，跟Windows下的函数Hook如出一辙
  descriptor.value = function (...args: any[]) {
    const startTime = process.hrtime.bigint();
    // 在原来得This上下文调用原始函数
    const result = originalMethod.apply(this, ...args);
    const endTime = process.hrtime.bigint();
    // 计算耗时
    console.log(`Method ${propertyKey} took ${endTime - startTime} ns`);
    // 返回结果
    return result;
  };
}

const greeter = new Greeter("world");
console.log(greeter.greet());
```

### PropertyDecorators

属性装饰器，主要用于属性得Metadata信息标记，一个经典案例：JSON 序列化字段过滤

```ts
const exposeFieldsMap = new Map();
const excludesFieldsMap = new Map();

function expose() {
  return function (target: any, propertyKey: string) {
    // 先获取类的字段标记，如果没有则创建
    let fields = exposeFieldsMap.get(target.constructor);

    if (!fields) {
      fields = [];
      exposeFieldsMap.set(target.constructor, fields);
    }

    fields.push(propertyKey);
  };
}

function excludes() {
  return function (target: any, propertyKey: string) {
    let excludesFields = excludesFieldsMap.get(target.constructor);

    if (!excludesFields) {
      excludesFields = [];
      excludesFieldsMap.set(target.constructor, excludesFields);
    }

    excludesFields.push(propertyKey);
  };
}

class SerializableWhitelist {
  [k: string]: any;
  toJSON() {
    // 通过当前实例的构造函数获取字段标记
    const fields = exposeFieldsMap.get(this.constructor) ?? [];
    // 根据字段标记进行属性过滤然后返回
    const result: Record<string, any> = {};

    for (const field of fields) {
      result[field] = this[field];
    }

    return result;
  }
}

class SerializableBlacklist {
  [k: string]: any;
  toJSON() {
    // 通过当前实例的构造函数获取字段标记
    const fields = excludesFieldsMap.get(this.constructor) ?? [];
    // 根据字段标记进行属性过滤然后返回
    const result: Record<string, any> = {};

    for (const field of Object.keys(this)) {
      if (fields.includes(field)) continue;
      result[field] = this[field];
    }

    return result;
  }
}

class User1 extends SerializableWhitelist {
  // 添加字段标记，会被序列化
  @expose()
  username = "Tom";
  // 没有被标记的属性，不会被序列化
  password = "123456";
}

class User2 extends SerializableBlacklist {
  @excludes()
  username = "Tom"; // 排除字段，不会被序列化
  password = "123456";
}

const user1 = new User1();
const user2 = new User2();
console.log(JSON.stringify(user1)); // {"username":"Tom"}
console.log(JSON.stringify(user2)); // {"password":"123456"}
```

### ParameterDecorators

参数装饰器，主要作用：校验、依赖注入；还是挺有用的，后续学习。

## 关于装饰器的 Target 参数

指向：TS 装饰器中的 target，指向“被装饰对象所属的那个对象” 它的具体指向取决于你装饰的是 类、方法、属性、访问器、参数。

1. 类装饰器：target 指向被装饰的类的构造函数(target === Greeter)
2. 普通方法装饰器：target 指向被装饰的类的原型对象或者说是实例的原型对象(target === Greeter.prototype)
3. 静态方法装饰器：target 指向被装饰的类的构造函数，与类装饰器一致(target === Greeter)
4. 普通属性装饰器：target 指向被装饰的类的原型对象(target === Greeter.prototype)
5. 静态属性装饰器：target 指向被装饰的类的构造函数(target === Greeter)

## 特殊类型

```ts
// 声明时可可以使用可选参数的形式来声明元组
type StringNumberBooleans = [string, number, ...boolean[]];
type StringBooleansNumber = [string, ...boolean[], number];
type BooleansStringNumber = [...boolean[], string, number];
```

上面这可选参数接受任意数量的参数，但是注意，运行时的数组或元组解构接收剩余参数时，只能写在最后：

```ts
const [a, ...b] = [1, 2, 3]; // ok
const [...a, b] = [1, 2, 3]; // error
const [a, ...b, c] = [1, 2, 3]; // error
```

## 保持不变性 readonly

设计API时，如果使用元组类型，尽可能使用readonly来声明元组，从而保证元组不变性，因为大部分场景下我们都不应该修改元组中的元素，这也符合函数式编程的思想：

```ts
// 使用readonly来声明元组
function doSomething(tuple: readonly [string, number]) {
  tuple[0] = "hello"; // 意外修改元组元素，会报错
}
```

另外，如果我们不生命只读元组的话，当外界传入的是只读元组，则会导致类型错误：

```ts
let point = [3, 4] as const;

function distanceFromOrigin([x, y]: [number, number]) {
  return Math.sqrt(x ** 2 + y ** 2);
}

// error 类型错误，只读类型元组无法赋值给可编辑元组，类似Rust中的不可变引用复制给可变引用一样，无法保证数据完整性
distanceFromOrigin(point);
```

因此API设计时最好使用readonly来声明元组。

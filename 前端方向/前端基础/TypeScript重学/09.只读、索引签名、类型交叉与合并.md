## 只读属性

TypeScript 在检查两种类型是否兼容时，不会考虑它们的属性是否为 readonly ，因此 readonly 属性也可能通过别名而改变

```ts
interface Person {
  name: string;
  age: number;
}

interface ReadonlyPerson {
  readonly name: string;
  readonly age: number;
}

let writablePerson: Person = {
  name: "Person McPersonface",
  age: 42,
};

// works
let readonlyPerson: ReadonlyPerson = writablePerson;

console.log(readonlyPerson.age); // prints '42'
writablePerson.age++;
console.log(readonlyPerson.age); // prints '43'
// Cannot assign to 'age' because it is a read-only property.
readonlyPerson.age++;
```

### 只读数组

我们没有任何手段创建只读数组，只需要将普通数组赋值给 ReadonlyArray 类型即可，这是纯粹的TS类型检查，JS层面并没有任何实质性的体现，也就没有可用的构造函数可用了。

```ts
const roArray: ReadonlyArray<string> = ["red", "green", "blue"];
// Property 'push' does not exist on type 'readonly string[]'.
roArray.push("hello");
```

注意，只读数组与普通数组是不可以进行双向赋值的，只能进行安全赋值（普通数组赋值给只读数组）：

```ts
// 只读的普通数组，变量只读，但是数组并非只读，是可以修改的
let x: readonly string[] = [];
let y: string[] = [];
x = y; // ok
y = x; // error：The type 'readonly string[]' is 'readonly' and cannot be assigned to the mutable type 'string[]'.(4104)

// 另外：readonly string[] 与 ReadonlyArray<string> 是等价的，没有区别
```

## 索引签名

索引签名一般用于描述对象中任意属性的类型，对于未知属性比较有用，基础语法：

```ts
interface SomeType {
  [index: string]: any;
}
```

有以下几个需要注意的地方：

1. 索引签名的索引类型只能是 string、number、symbol、模板字符串模式以及仅由这些类型组成的联合类型
2. 索引签名与其他属性签名可以共存，只要索引签名的类型与其他属性签名的类型兼容即可：索引属性类型 > 属性类型
3. number类型的索引签名类型与string类型的索引签名共存时，number签名的索引类型必须是string签名的索引类型的子类型（因为在js中使用对象在使用属性访问器时，如果传入数字的话会被转换为string进行索引，TS选择了与JS引擎保持一致的处理方式：obj[100] => obj["100"]）

```ts
// 与其他属性共存，类型需要保持兼容
interface NumberOrStringDictionary {
  [index: string]: number | string;
  length: number; // ok, length is a number
  name: string; // ok, name is a string
}

// 使用模板字符串创建特定模式的属性，多种类型的索引共存
type DataMap = {
  [key: `data-${string}`]: number;
  [key: string]: number | string;
  [key: number]: number;
};

const obj: DataMap = {
  "data-a": 114514, // ok
  "data-b": "114514", // error
  aaa: 123, // ok
  1: 123, // ok
};

// 搭配read-only 使用
interface ReadonlyStringArray {
  readonly [index: number]: string;
}
```

## 未知的属性

对于将来可能出现的未知属性我们可以使用索引签名来处理，但是不要使用any，而是使用unknown：

```ts
type Options = {
  name: string;
  expires?: number;
  // 将来可能出现的配置选项预留
  [other: string]: unknown;
};
```

## Interface 覆盖、继承扩展

继承：

```ts
interface Colorful {
  color: string;
}

interface Circle {
  radius: number;
}

interface ColorfulCircle extends Colorful, Circle {
  // add new properties here
}

const cc: ColorfulCircle = {
  color: "red",
  radius: 42,
};
```

覆盖，Interface 如果在同一个命名空间内存在同名定义，TS会尝试合并，将所有Interface定义的属性合并，如果属性类型冲突，则报错：

```ts
interface A {
  color: string;
}
interface A {
  title: string;
}

// 两个定义均生效，a必须具备color属性和title属性
const a: A = {
  color: "red",
  title: "hello world",
};
```

## 组合交叉类型

```ts
interface Colorful {
  color: string;
}
interface Circle {
  radius: number;
}

type ColorfulCircle = Colorful & Circle;
```

## 交叉与继承的区别

主要区别在于属性类型冲突时的处理方式，interface在遭遇属性类型冲突时，会报错，而不是强行合并：

```ts
interface A {
  name: string;
}
interface B {
  name: number | string;
}
// Interface 'C' cannot simultaneously extend types 'A' and 'B'.
// Named property 'name' of types 'A' and 'B' are not identical.
interface C extends A, B {}
```

但是对于交叉类型，TS不会报错，而是强行合并，类型不见容属性由于无法求解出类型的最大公约数，因此类型会被设置为never：

```ts
type A = { name: string; age: string };
type B = { name: string; age: number };
type C = A & B;

const c: C = {
  name: "Alice",
  // Type 'number' is not assignable to type 'never'.(2322)
  // input.tsx(1, 26): The expected type comes from property 'age' which is declared here on type 'C'
  age: 18,
};
```

上面属性age合并之后的类型为never，因此，属性age无法被任何类型赋值

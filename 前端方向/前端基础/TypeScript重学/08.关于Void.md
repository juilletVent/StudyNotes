## 历史悠久的 void 0

在 es3 时期（1999年），undefined 并不是一个关键字或者保留字，因此你可以写出如下这种放到今天看起来诡异的代码：

```typescript
var undefined = true;

if (undefined) {
  console.log("惊不惊喜，意不意外~");
}
```

但是 undefined 又确实是一个值，因此获取真正的undefined，需要一个安全的途径，在js中 void 是一个一元操作符，语法为：void expression，它计算表达式的值，并始终返回 undefined

因此，通过开发者的大量实践，void 0 就是获取真正的undefined的最佳实践，然后沿用至今。

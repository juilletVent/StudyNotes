## `Awaited<Type>`

递归地解包 Promise 类型

```ts
type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T;

type TypeA = Awaited<boolean | Promise<number>>;
type TypeB = Awaited<Promise<Promise<Promise<number>>>>;
```

## `Partial<Type>`

构造一个类型，并将 Type 的所有属性设置为可选

```ts
type Partial<T> = { [K in keyof T]?: T[K] };

type A = { a: number; b: string };
type OptionalA = Partial<A>; // { a?: number; b?: string }
```

## `Required<Type>`

与`Partial<Type>` 相反，将所有可变属性设置为必选

```ts
// 注意语法：`-?`
type Required<T> = { [K in keyof T]-?: T[K] };
```

## `Readonly<Type>`

将所有属性设置为只读

```ts
type Readonly<T> = { +readonly [K in keyof T]: T[K] };
```

## `Record<Keys, Type>`

根据Keys 创建一个对象类型，对象类型中的属性类型为Type

```ts
type Record<K extends keyof any, T> = {
  [P in K]: T;
};
```

## `Pick<Type, Keys>`

从Type 中选择某些属性

```ts
type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};
```

## `Omit<Type, Keys>`

从Type 中排除某些属性

```ts
type Omit<T, K extends keyof T> = {
  // 使用 as 语法限定 P 类型排除于 K 内
  [P in keyof T as P extends K ? never : P]: T[P];
  // 官方使用 Exclude 语法，更加简洁，本质上是等效的
  [P in keyof T as Exclude<P, K>]: T[P]
};
```

## `Exclude<Union, Excluded>`

从 Union 中排除某些成员，这里用到了联合类型分发，可能不是很好理解

```ts
// 因为 T 是裸类型，所以回进行联合类型分发，回将T进行拆分，分别进行判断，然后将所有结果联合返回
// Exclude<'a' | 'b', 'b'>
// 判断：'a' extends 'b' ? never : 'a' => 'a'
// 判断：'b' extends 'b' ? never : 'b' => never
// 最终返回：'a' | never => 'a'
type Exclude<T, K> = T extends K ? never : T;
```

## `Extract<Union, Extracted>`

从 Type 中提取所有可赋值给 Union 联合成员来构造类型，与 Exclude 是相反的，同样利用联合类型分发

```ts
type Extract<T, K> = T extends K ? T : never;
```

## `NonNullable<Type>`

从 Type 中排除 null 和 undefined 来构造类型

```ts
// 旧版实现，还好理解，不赘述了
type NonNullable<T> = T extends null | undefined ? never : T;
// 新版实现：利用了一个TS特殊约定：空对象是任何类型的父类型（null、undefined除外）
// 搭配上联合类型分发，就可以过滤掉 null 和 undefined了
// 比如：NonNullable<string | null | undefined>
// 拆分：string | null | undefined
// 判断：string & {} => string
// 判断：null & {} => never
// 判断：undefined & {} => never
// 最终返回：string | never | never => string
type NonNullable<T> = T & {};
```

## `Parameters<Type>`

从函数类型中获取参数表类型

```ts
// 借助infer推导参数表类型
type Parameters<T extends (...args: any) => void> = T extends (
  ...args: infer R
) => void
  ? R
  : never;
```

## `ConstructorParameters<Type>`

从构造函数类型中获取参数表类型，原理同上

```ts
type ConstructorParameters<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: infer P) => any ? P : never;
```

## `ReturnType<Type>`

从函数类型中获取返回值类型

```ts
type ReturnType<T extends (...args: any) => any> = T extends (
  ...args: any
) => infer R
  ? R
  : any;
```

## `InstanceType<Type>`

从构造函数类型中获取实例类型

```ts
type InstanceType<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: any) => infer R ? R : any;
```

## `NoInfer<T>`

阻止类型参数 T 的类型推导

```ts
function createStreetLight<C extends string>(
  colors: C[],
  // 默认情况下，TS将尝试推导出两个参数的类型最大公约数，但是此处 defaultColor 应该是中包含在颜色列表中的，不应该参与类型推导
  defaultColor?: NoInfer<C>,
) {
  // ...
}
createStreetLight(["red", "yellow", "green"], "red"); // OK
createStreetLight(["red", "yellow", "green"], "blue"); // Error
```

## `ThisParameterType<Type>`

从函数类型中获取 this 参数类型，如果函数没有 this 参数，则返回 unknown

```ts
type ThisParameterType<T> = T extends (this: infer U, ...args: never) => any
  ? U
  : unknown;
```

## `OmitThisParameter<Type>`

从 Type 中移除 this 参数。如果 Type 没有显式声明 this 参数，则结果为 Type 本身。否则，将从 Type 创建一个没有 this 参数的新函数类型

```ts
type OmitThisParameter<T> =
  // 检查类型是否有显示This类型，如果没有，直接返回
  unknown extends ThisParameterType<T>
    ? T
    : // 推导剩余参数表，如果失败，直接返回
      T extends (...args: infer A) => infer R
      ? (...args: A) => R
      : T;
```

## `ThisType<Type>`

_Tips: noImplicitThis 必须启用才能使用ThisType_

此工具不会返回转换后的类型，使用它可以在特定上下文明确指定this类型，案例：

```ts
type ObjectDescriptor<D, M> = {
  data?: D;
  // 标记 methods 下的所有方法的 this 类型为 D & M
  methods?: M & ThisType<D & M>;
};

function makeObject<D, M>(desc: ObjectDescriptor<D, M>): D & M {
  return {
    ...desc.data,
    ...desc.methods,
  } as D & M;
}

const obj = makeObject({
  data: {
    x: 0,
    y: 0,
  },

  // methods 下所有方法的this 类型为 D & M，可以访问 data 的属性
  methods: {
    move(dx: number, dy: number) {
      this.x += dx;
      this.y += dy;
    },
  },
});

console.log(obj);
obj.move(1, 2);
console.log(obj);
```

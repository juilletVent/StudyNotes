## 类型谓词

我们可以借助类型谓词来缩小类型的范围，从而在运行时避免类型错误。下面是一个典型的案例：

```typescript
function isFish(pet: Fish | Bird): pet is Fish {
  return (pet as Fish).swim !== undefined;
}
```

语法格式：`parameterName is Type`，其中 `parameterName` 是一个参数名，`Type` 是一个类型。类型谓词的返回值是一个布尔值，表示参数是否为指定的类型，配合if完成类型缩小。

```typescript
function getFood(pet: Fish | Bird) {
  if (isFish(pet)) {
    pet; // 类型缩小为 Fish
    return "fish food";
  } else {
    pet; // 类型缩小为 Bird
    return "bird food";
  }
}
```

## 断言函数

与类型谓词非常相似的一种类型收窄机制，区别在于，断言函数在断言失败时会抛出一个错误，而不是返回false

第一种语法：

```typescript
function assertString(value: unknown): asserts value is string {
  if (typeof value !== "string") {
    throw new Error("Not string");
  }
}

let val = Math.random() > 0.5 ? 1 : "string";
assertString(val);
// 类型收窄为string，如果类型不匹配则会抛出错误
console.log(val);
```

第二种语法：

```typescript
function assert(condition: any, msg?: string): asserts condition {
  if (!condition) {
    throw new Error(msg);
  }
}

let val = Math.random() > 0.5 ? 1 : "string";
// 此处使用第二种语法，如果类型不匹配，则会抛出错误
assert(typeof val === "string", "类型错误，只接受string类型");
// 类型收窄为string
console.log(val);
```

_Tips: 两种语法主要区别在于第一种语法用于类型收窄，第二种语法用于控制流守卫，如果是判断某个东西是什么类型，用第一种，如果是判断某个东西是否符合某个条件，用第二种_

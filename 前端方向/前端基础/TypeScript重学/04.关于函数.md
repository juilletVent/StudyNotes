## 函数签名

函数签名有两种类型：

```typescript
// 第一种
function add(x: number, y: number): number;
// 第两种
type Add = {
  (x: number, y: number): number;
};
```

如果需要定义一个带属性的函数：

```typescript
type Add = {
  (x: number, y: number): number;
  description: string;
};

const add: Add = (x, y) => x + y;
add.description = "This function adds two numbers.";

console.log(add(1, 2));
console.log(add.description);
```

## 构造函数

```typescript
type PersonConstructor = {
  // 使用new标记，表示这是一个构造函数
  new (name: string, age: number): Person;
```

## 回调函数中的可选参数

```typescript
function forEach<T>(
  arr: T[],
  callback: (item: T, index?: number) => void,
): void {
  for (let i = 0; i < arr.length; i++) {
    callback(arr[i], i);
  }
}
```

上面这种做法，编写人的意图可能是允许下面的调用方式：

```typescript
forEach([1, 2, 3], (item) => {
  // do something...
});
forEach([1, 2, 3], (item, index) => {
  console.log(index.toFixed(2)); // 可能为空
});
```

但是从TS的角度来看，callback的index参数是可选的，因此，哪怕内部实现保证了index不可能为空，但是TS还是会给出错误提示，

因此：**编写回调函数类型时，除非打算在不传递参数的情况下调用该函数，否则切勿编写可选​​参数。**

## 函数重载

准则一：函数重载所声明的函数签名必须与函数实现所声明的函数签名兼容，否则无法他过编译。

准则二：尽可能优先选择联合类型的参数，而不是重载类型。

比如下面的案例：

```typescript
function fn(x: number): boolean;
function fn(x: string): string;

function fn(x: string | number): boolean | string {
  if (typeof x === "string") return "x";
  return true;
}
// 错误：Type 'number' is not assignable to type 'string | number'.
const res = fn(Math.random() > 0.5 ? "aaa" : 111);
```

这个案例使用重载，精确定义了不同入参情况下的函数签名，精确描述了对应场景的返回值类型，但是如果只是入参类型不同，返回值类型相同，则没必要使用重载：

```typescript
// 两个类型分支，返回类型都是为 number，没有必要使用函数重载签名
function len(x: any[] | string) {
  return x.length;
}
```

## This 类型

为this指定类型，使用场景并不多

### 为回调指定this类型

```typescript
type User = {
  name: string;
};

interface Collection<T> {
  each(fn: (this: T) => void): void;
}

class UserCollection implements Collection<User> {
  users: User[] = [];
  // 此处的fn声明表述了fn只能在User实例中调用，因此，在Fn中，可以直接使用this
  each(fn: (this: User) => void) {
    for (const user of this.users) {
      // 必须使用call/apply来确定this的指向
      fn.call(user);
    }
  }
}

const collection = new UserCollection();

collection.each(function () {
  // 由于确定了this的类型为User，因此，this.name是允许的
  console.log(this.name);
});
```

### 为对象方法指定this类型

```ts
interface User {
  name: string;
  rename(this: User, name: string): void;
}
```

### 禁止使用this

如果希望回调中不允许使用this，这可以使用这种方式禁用

```ts
interface Button {
  onClick(handler: (this: void) => void): void;
}
```

### 链式API

```ts
interface Builder {
  setName(this: Builder, name: string): this;
}
```

## 函数返回类型 void

```ts
type voidFunc = () => void;

const f1: voidFunc = () => {
  return true;
};
const f2: voidFunc = () => true;
const f3: voidFunc = function () {
  return true;
};
```

上面代码不会报错，并且f1、f2、f3的返回值类型都是void，ts会直接忽略返回值，哪怕实际返回的是boolean，为什么会这样实现？是因为对外来说，函数类型是没有返回值的，就算函数实现返回了值，外部也不会使用，因此可以安全丢弃，实际案例：

```ts
const src = [1, 2, 3];
const dst = [0];
// push 会返回el，但是forEach 压根儿不关心返回值，所以TS保持void返回类型就很合理
src.forEach((el) => dst.push(el));
```

相对于：() => true 是 () => void 的子类型

## 基础条件类型用法

```ts
interface Animal {
  live(): void;
}
interface Dog extends Animal {
  woof(): void;
}

type Example1 = Dog extends Animal ? number : string; // number
type Example2 = RegExp extends Animal ? number : string; // string
```

语法：`SomeType extends OtherType ? TrueType : FalseType;`

当 extends 左侧的类型可以赋值给右侧的类型时，你将得到第一个分支（“真”分支）中的类型；否则，你将得到后一个分支（“假”分支）中的类型。

> 条件类型的强大之处在于它们与泛型结合使用

案例：

```ts
interface IdLabel {
  id: number /* some fields */;
}
interface NameLabel {
  name: string /* other fields */;
}

// 不使用条件类型，需要使用函数重载来完成，类型多了，将会是灾难性的
function createLabel(id: number): IdLabel;
function createLabel(name: string): NameLabel;
function createLabel(nameOrId: string | number): IdLabel | NameLabel;
function createLabel(nameOrId: string | number): IdLabel | NameLabel {
  throw "unimplemented";
}

// 使用条件类型，可以更简洁的实现
type NameOrId<T extends number | string> = T extends number
  ? IdLabel
  : NameLabel;
function createLabel<T extends number | string>(idOrName: T): NameOrId<T> {
  throw "unimplemented";
}

// 实际使用时：
let a = createLabel("typescript"); // NameLabel
let b = createLabel(2.8); // IdLabel
let c = createLabel(Math.random() ? "hello" : 42); // IdLabel | NameLabel
```

另外一个例子：

```ts
// 约束泛型不需具备某些属性
type MessageOf<T> = T extends { message: unknown } ? T["message"] : never;

interface Email {
  message: string;
}
interface Dog {
  bark(): void;
}

type EmailMessageContents = MessageOf<Email>; // string
type DogMessageContents = MessageOf<Dog>; // never
```

## Infer 推断

下面这个工具类型使用类型约束来提取数组的元素类型，这种操作在TS中非常常见，以至于TS为这个行为增加了专门的语法，使用 infer

```ts
// 不使用infer，我们需要使用索引访问类型来获取数组元素类型
type Flatten<T> = T extends any[] ? T[number] : T;
// 使用infe，Infer 语法，将数组元素类型提取出来，我们可以直接使用Item类型
type Flatten<T> = T extends Array<infer Item> ? Item : T;
```

更多Infer构建的工具类型：

```ts
// 返回值推断，函数参数逆变，必须使用最宽松的匹配，也就是never[]，这里的约束规则由于参数是逆变模式，所以不能使用unknown[]
type GetReturnType<T> = T extends (...arr: never[]) => infer Return
  ? Return
  : never;
type Fn = (val: string[]) => Promise<void>;
// 借助工具类型获取函数签名中的返回值类型
type RetType = GetReturnType<Fn>; // Promise<void>
```

**重点：如果函数具备多个重载声明，工具函数推断时，始终使用最后一个重载声明进行推断，因此声明顺序将会直接影响推断结果**

```ts
declare function stringOrNum(x: string): number;
declare function stringOrNum(x: number): string;
declare function stringOrNum(x: string | number): string | number;

type T1 = ReturnType<typeof stringOrNum>; // 调整前面的重载函数声明顺序将会影响最终类型推导结果，这与函数签名的宽松性没有关系
```

## 联合类型的条件分发

```ts
type ToArray<T> = T extends any ? T[] : never;
// 将某个联合类型转化为数组
type StrArrOrNumberArr = ToArray<string | number>; // string[] | number[]
```

你可能认为这合理也可能认为这不合理，那么如果我希望的到的结果是：`(string | number)[]`，阁下又该如何应对呢？

TS：有的，有的，兄弟，有的！

```ts
// 下面这些款式都可以，任选其一即可
type ToArray<T> = T[] extends any ? T[] : never;
type ToArray<T> = T[] extends any[] ? T[] : never;
type ToArray<T> = [T] extends any ? T[] : never;
type ToArray<T> = [T] extends any[] ? T[] : never;
type ToArray<T> = [T] extends [any] ? T[] : never;

type StrArrOrNumberArr = ToArray<string | number>; // (string | number)[]
// 下面这个不行哈
// type ToArray<T> = T[] extends [any] ? T[] : never;
```

**重点：只要条件类型不是裸类型，就不会发生联合类型分发**

```ts
// 这里的泛型 T 只要不是裸类型，就不会进行分发
type ToArray<T> = T extends any ? T[] : never;
// 如果泛型类型是 T[] [T] Promise<T> 则不会进行分发，比如：
type ToArray<T> = Promise<T> extends any ? T[] : never; // ToArray<string | number> => (string | number)[]
```

### 典型应用场景

```ts
type A = {
  a: string;
  common: string;
};

type B = {
  b: number;
  common: string;
};

// 收集公共Key：keyof 是操作符，不是条件类型，不会分发，因此提取公共字段保证可用性
type CommonKeys = keyof (A | B);
// 收集所有Key：条件类型，会进行联合体类型分发，因此最终得到：a | b | common
type GetAllKeys<T> = T extends any ? keyof T : never;
type AllKeys = GetAllKeys<A | B>;
```
